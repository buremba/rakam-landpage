One-click analytics backend

Rakam API is an open-source analytics backend with more then 100 endpoints. It exposes a RESTFul API with more than 100 endpoints for both collecting and analyzing data.

We use Swagger for defining API endpoints so Rakam API can be integrated into your current infrastructure. You can also develop custom modules and implement new endpoints and features depending on your needs.

You can directly connect your Rakam API via RESTFul API or Rakam API client library for your favorite programming language. Our SDKs are also built on top of Rakam API so you can easily set your Rakam API and send your data directly to your Rakam API Cluster.

Fast and scalable

A single Rakam API server can handle more than 50K events concurrently, it's based on Java 8, Netty and Jackson. We use our custom JSON parsers in order to use lower memory footprint and CPU, Netty allows us to handle thousands of requests concurrently without any problem. You can also scale your Rakam API by adding new nodes to the cluster, we provide extra tools for scaling your Rakam API cluster using your favorite cloud provider.

Rich Query API

We provide you the most common analysis features required for an analytics service. Event analytics is easy with event explorer and real-time features and behavioral analytics queries can be run via funnel and retention features.
You can also run SQL queries via Query API, integrate your custom databases and remote files and merge data to build your own reports.
Since Rakam stores the raw data, we provide some some features to be able to create summary tables, OLAP cubes, stream processing rule for pre-aggregating the data and accessing the the data faster.
See analyze data section in documentation.

Customizable ETL

The ETL process in Rakam API is completely transparent. We have both native and custom event mappers, you can write modules that implements native event mappers in Java for stateful and fast ETL tasks and inject it to your Rakam API event collection process.
However writing Java modules are not an easy task so we have custom event mappers with Javascript embedded in Rakam. You can basically write Javascript mappers and let Rakam to invoke them when a new event arrives. They may enrich your event data or sanitize bad inputs, it's up to you.

In fact, in most cases you won't need to write event mappers because Rakam provides a set of event mappers for different use-cases. However, it's important for us to provide you the ways to enrich and sanitize your event data so that you can embed your event pipeline inside Rakam without hassle.
See event mappers section in documentation.

Collect data from anywhere

Rakam is data agnostic, you can send any kind of event data-set as long as it's immutable. You can just use the SDKs in order to ingegrate your application with Rakam easily or use the RESTFul API. We also provice embedded scheduled tasks, webhooks and some high level features for you to collect data from third party services. You can basically embed your scripts in Rakam and let it collect data for you. No cron-jobs, no manual process and no extra work after you setup the tasks.
See collecting events section in documentation.

SELECT total_order_amount / fb.spent_amount_total AS revenue_ads_ratio FROM (
    SELECT cast(_time AS date) day, sum(total_amount) AS total_order_amount FROM (
        SELECT _user, total_amount FROM collection.transaction
           JOIN mysqldb.users ON (users.id = transaction._user)
       UNION ALL SELECT user_id, total_amount FROM remotefile.offlinesales
    ) WHERE _user IN (SELECT user_id FROM materialized.gold_users WHERE clearbit_job = 'Doctor')
) orders JOIN fb_advertisement fb ON (orders.day = fb.day)
